## 2024-05-18 - Avoid onclick handlers on non-interactive elements without ARIA/keyboard support\n**Learning:** The options page previously used `<span>` elements with `onclick` handlers for renaming groups. This creates a significant accessibility issue because screen readers and keyboard users cannot easily interact with them. If a non-interactive element like a `<span>` is used for an action, it must have `role="button"`, `tabindex="0"`, a descriptive `aria-label`, and a keyboard event listener for `Enter` and `Space`. Alternatively, it is better to use a semantic HTML element like `<button>`.\n**Action:** Always ensure that interactive elements are either semantic HTML (like `<button>`) or have the appropriate ARIA roles, tabindex, and keyboard event handlers.
